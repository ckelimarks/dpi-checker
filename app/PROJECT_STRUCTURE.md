```
    └── CreativeFabricaCTA
```

### Data Flow
```
User uploads file
    ↓
FileUpload validates & passes File object
    ↓
App.jsx calls validateFile(file, platform)
    ↓
validation.js processes file:
  - Extract dimensions (Canvas API)
  - Calculate DPIs at 4 sizes
  - Validate against platform requirements
    ↓
Returns validation results object
    ↓
ValidationResults displays:
  - Overall status
  - DPITable (multi-size display)
  - CheckResult components (detailed checks)
  - CreativeFabricaCTA (context-aware)
```

## Key Files & Their Responsibilities

### 1. `/src/App.jsx` (Main Orchestrator)
**Responsibilities:**
- Manage application state (platform, results, loading)
- Coordinate between components
- Handle file upload callback
- Render appropriate UI state (upload/loading/results)

**State:**
- `platform` - Selected platform (printful/printify/general)
- `results` - Validation results object
- `isAnalyzing` - Loading state

### 2. `/src/utils/validation.js` (Core Logic)
**Responsibilities:**
- Define platform requirements
- Load image and extract dimensions
- Calculate effective DPI at multiple print sizes
- Validate against platform-specific rules
- Generate shareable URLs

**Key Functions:**
- `validateFile(file, platform)` - Main validation function
- `loadImage(file)` - Extract dimensions from PNG/JPG/SVG
- `calculateDPIsAtSizes()` - Calculate DPI for each print size
- `generateShareableURL()` - Create shareable URL

### 3. `/src/components/ValidationResults.jsx` (Results Orchestrator)
**Responsibilities:**
- Display overall pass/fail status
- Render DPI table
- Show detailed check results
- Display Creative Fabrica CTA
- Handle share functionality
- Provide "Validate Another File" button

**Props:**
- `results` - Validation results object
- `platform` - Selected platform
- `onReset` - Callback to reset state

### 4. `/src/components/DPITable.jsx` (Multi-Size Display)
**Responsibilities:**
- Display DPI results for 4 print sizes
- Show status indicators (✅ ⚠️ ❌)
- Provide educational explanation

**Props:**
- `dpiResults` - Array of DPI calculation results

### 5. `/src/components/FileUpload.jsx` (Upload Interface)
**Responsibilities:**
- Drag-and-drop functionality
- Click-to-browse fallback
- File type validation
- File size validation
- Error display

**Props:**
- `onFileUpload` - Callback when valid file selected
- `isAnalyzing` - Disable during processing

### 6. `/src/components/PlatformSelector.jsx` (Platform Selection)
**Responsibilities:**
- Display platform options
- Show requirements for each platform
- Visual selection state

**Props:**
- `selectedPlatform` - Current platform
- `onPlatformChange` - Callback when platform changes

### 7. `/src/components/CheckResult.jsx` (Reusable Check Display)
**Responsibilities:**
- Display individual validation check
- Show pass/fail status
- Render prescriptive content

**Props:**
- `title` - Check name
- `pass` - Pass/fail status
- `allPass` - Optional flag for warning state
- `children` - Prescriptive content

### 8. `/src/components/CreativeFabricaCTA.jsx` (Marketplace CTA)
**Responsibilities:**
- Display context-aware CTA
- Different messaging for pass/fail states
- Link to Creative Fabrica marketplace

**Props:**
- `overallPass` - Determines pass vs fail messaging

## No External Validation Libraries

The core validation logic is implemented in vanilla JavaScript using browser APIs:
- File API (file reading)
- Canvas API (image dimension extraction)
- Clipboard API (copy to clipboard)
- URL API (shareable URLs)

This keeps the bundle small and removes external dependencies.
